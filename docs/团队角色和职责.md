## 📋 **工作规则和流程**

### **任务接收规则**:
1. **确认接收** - 收到任务后必须回复"收到任务"
2. **阅读文档** - 仔细阅读对应的任务文档
3. **提出疑问** - 有不清楚的地方立即询问架构师
4. **开始执行** - 按时间安排开始工作

### **进度汇报规则**:
1. **定时汇报** - 每30分钟汇报一次进展
2. **格式统一** - 使用标准格式汇报
3. **问题及时** - 遇到问题立即求助
4. **完成通知** - 任务完成后立即通知

### **协作规则**:
1. **相互支持** - 窗口间可以相互协助
2. **信息共享** - 重要信息及时共享
3. **冲突解决** - 技术分歧由架构师裁决
4. **质量优先** - 代码质量优于开发速度

### **质量标准**:
1. **代码审查** - 关键代码需要架构师审查
2. **测试验证** - 功能完成后必须测试验证
3. **文档更新** - 重要变更需要更新文档
4. **最佳实践** - 遵循团队技术规范
